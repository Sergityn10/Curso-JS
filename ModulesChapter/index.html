Chapter 11: Modules

El objetivo de todo programador es crear un código limpio y con el menor número de dependencias posibles. Para ello, se empezarón a organizar los códigos en módulos, que son archivos separados que contienen código JavaScript. Cada módulo se le debe identificar 2 partes fundamentales, característicos de estos:

  1. Sus dependencias: Se trata de otras partes de código, las cuales, el funcionamiento del módulo depende de ellos.
  2. Su interfaz: Se trata del cuerpo del módulo. Es decir, el código en sí.

Los módulos es una buena forma de no repetir código y, por lo tanto, cumplir con la regla DRY. Además, cada módulo debe de especificar aquellos parámetros, métodos y funciones que se van a querer "exportar" para que el sistema de módulos pueda recoger dicha vinculación correctamente. La forma de importar y exporta depende de cada sistema de modulos que se quiera utilzar.

Al principio, este sistema de módulos no existían, pero con el paso del tiempo, JS ha ido incorporando formas de modulado y los programadores, también, se debían apañar como sea. Por esta razón, a lo largo del tiempo, han existido distintas formalidades de establecer un sistema de módulos:

Empaquetadores

Los empaquetadores son herramientas que permiten agrupar varios módulos en un solo archivo. Esto se hace para que el navegador pueda cargar el código de manera más rápida y eficiente. Algunos de los empaquetadores más populares son Webpack y Rollup.

Sistema de paquetes

JavaScript ofrece un sistema de publicación e instalación de paquetes llamada NPM. NPM ofrece una infinidad de paquetes, tanto útiles, como inútiles, que puedes utilizar instalandolos en tu proyecto como dependencias. Para instalarlo se deberá especificar, escribiendo en tu consola de comandos el siguiente script:

npm install "paquete"

Si desea instalarlo de forma global en tu ordenador, deberás añadir el comando -g, como se ve en el siguiente ejemplo:

npm install -g "paquete"
